Chapter 1.3: Scope and The Root Table

Where your variables and functions "live" is called their scope.

  • Variables created with local only exist inside the curly braces { } where they are defined. This is a local scope.
  • Variables created in a table exist within that table's scope.
  • Variables not inside any function or table live in the global scope.

In Squirrel, the global scope is just one giant table called the root table. When you write myVar <- "hello" at the top level of your file, you are creating a slot in the root table.

You can explicitly access the root table from anywhere using the :: operator.

// This creates a slot 'globalMessage' in the root table.
globalMessage <- "This is a global message."

function SomeFunction()
{
// This creates a variable that only exists inside this function.
local localMessage = "This is a local message."

// To access the global variable from inside a function, we must use ::
printl(::globalMessage)
}

SomeFunction()
// printl(localMessage) // This would cause an error! localMessage doesn't exist here.

Best Practice: Always use local for temporary variables inside functions. Only use global variables (::) when you absolutely need data to be accessible from everywhere. Overusing them can lead to messy, hard-to-debug code.
