Chapter 2.1: The logic_script Entity & Scope

The simplest way to run a script file is with a logic_script entity. This is an entity you place in your map using the Hammer editor.

When the map loads, the engine will find your logic_script and automatically run the code inside its specified script file. The code runs from top to bottom, one time. This is great for setting things up when the map starts.

Scope Visualization

Each logic_script (and each entity script) gets its own unique table, or scope, inside the main Root Table. This is important: it means variables created in one script are not automatically visible to another.

[Root Table (Global Scope)]
[Scope for logic_script_A]
puzzle_A_counter <- 0
[Scope for logic_script_B]
puzzle_B_state <- "inactive"

logic_script_A cannot directly see puzzle_B_state. It would need to use global variables (::) or other advanced methods to communicate.
